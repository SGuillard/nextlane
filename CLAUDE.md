# Working protocol

Use this repository for the Nextlane Staff AI Engineer case study.

Before changing code:

- Read `README.md`.
- Read `AGENTS.md`.
- Read the relevant document in `docs/`.
- Keep the DMS slice small.

For any feature:

1. State the smallest product behavior to change.
2. Implement the vertical slice.
3. Add or update tests.
4. Update documentation when behavior changes.
5. Run verification.
6. Run the `grillme-with-docs` review workflow.

Priorities:

1. AI leverage and artifact quality.
2. Live day-2 extensibility.
3. Engineering judgment and delivery.
4. Communication and technical leadership.
